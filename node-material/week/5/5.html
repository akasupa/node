Week 5 - Testing

Excellent tests are critical to success in node.js. Due to JavaScript's dynamic nature, test support for refactoring and regressions is far more important than in non-dynamic languages. This week, we'll cover testing and writing unit tests with mocha , chai , sinon and finally gulp as a task runner.

Group Milestone - Implementation

The final weeks will be spent developing the app that was planned and designed by your team. First steps include identifying the activities, models, and libraries that will be used within your Node.js app. What are the API endpoints? Which data models will you need? Which interactions and functions require external libraries? Next, the team will collaborate on the implementation using Git as a version control system. Review the Collaborating on Projects with Git guide to get started.

Deliverable: (Before June 4th) Raise an issue on github once you've completed your first sprint. This week your group should:

  • Break down work for each team member to complete for this week into issues on Github assigned to a milestone.
  • Review the collaboration guide when starting your project
  • Setup and configure your data store (Mongo, Postgres, etc...) on Heroku
  • Design your database schema and generated model objects with initial fields
  • Create activities and fragments needed for early functionality.
  • Identify all core flows and work needed for your app's required stories to be functionally complete

Once your sprint is complete (before June 4th):

Required: Create an issue on Github (/cc @codepathreview @codepath) with a GIF walkthrough.

  • This requires the team to integrate their codebase and record a GIF that demonstrates that the app's core is functionally complete.
  • Graders will review your progress to ensure you have a functionally complete app this week
